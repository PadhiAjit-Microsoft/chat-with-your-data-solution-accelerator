## Summary

Completed the first `azd up` of CWYD v2 on the new tenant/subscription. Provisioning + build/push/deploy of all three container services **succeeded**, and the post-provision index + knowledge base seeded. But a live review of the running deployment found the **backend Container App crashes on startup**: it reads a Cosmos DB container during the FastAPI lifespan and gets `(Forbidden) ... blocked by your Cosmos DB account firewall`, so `/api/health` never responds. Root cause is that Cosmos DB + Storage come up with `publicNetworkAccess: Disabled` in the default public profile — logged as [BUG-0093](../bugs.md).

## Planned

- Repoint the azd env to the new subscription, verify quota, run the first `azd up`.
- Review the live deployment (health + grounding).
